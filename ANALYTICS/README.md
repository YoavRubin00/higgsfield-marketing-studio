## מה כלל בכל snapshot

### חובה
1. **תאריך snapshot** + תאריך snapshot הבא (30 יום קדימה)
2. **תמונת מצב כללית**: עוקבים, פוסטים, ממוצע engagement
3. **טבלת ביצועים** של כל הפוסטים מאז ה-snapshot הקודם (תאריך, סוג, נושא, לייקים, תגובות, שמירות)
4. **Top 3 / Bottom 3 פוסטים** + ניתוח קצר למה הם הצליחו/כשלו
5. **דפוסים חדשים שזוהו** (winning + failed)
6. **השוואה לחודש קודם** (delta בעוקבים, ב-engagement rate)
7. **המלצות אקשן** ל-30 הימים הבאים

### רשות
- צילומי מסך של פוסטים מובילים
- תגובות מעניינות (רעיונות לתוכן עתידי)
- ציטוטים של עוקבים (לפוסט עמוד 4 — סיפורים)

## תהליך עדכון

1. **שחזור נתונים מהפלטפורמה**: מהאינסטגרם בידני (insights) או דרך MCP אם זמין
2. **יצירת קובץ חדש**: לא לערוך קובץ ישן — תמיד לפתוח קובץ חדש לחודש החדש
3. **השוואה לחודש קודם**: לקרוא את ה-snapshot הקודם, לחשב delta
4. **עדכון `BRAND/content-pillars.md`** רק אם זוהו שינויים מהותיים בדפוסים (לא בכל חודש)
5. **עדכון `BRAND/voice.md`** רק אם hook חדש נכנס ל-top performers
6. **commit ל-git** עם הודעה: `feat(analytics): instagram baseline YYYY-MM`
